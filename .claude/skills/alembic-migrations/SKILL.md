# Alembic Migrations — DailyLoadout

## Commands

```bash
cd packages/api
poetry run alembic upgrade head              # apply all migrations
poetry run alembic revision --autogenerate -m "description"  # generate migration
poetry run alembic revision -m "description" # empty migration (manual)
poetry run alembic current                   # show current revision
poetry run alembic history                   # show migration history
poetry run alembic downgrade -1              # rollback one migration
```

Or via Makefile:
```bash
make api-migrate                             # alembic upgrade head
make api-migration msg="description"         # new autogenerated migration
```

## Naming Convention

- `create_{table}s_table` — new table
- `add_{column}_to_{table}s` — new column
- `add_idx_{table}s_{column}` — new index
- `alter_{table}s_{change}` — modify existing

## Schema Conventions

### Primary Keys (internal)
```python
id = sa.Column(sa.Integer, primary_key=True, autoincrement=True)
```

### Public ID (exposed to API)
```python
public_id = sa.Column(sa.String, unique=True, nullable=False, server_default=sa.text("gen_random_uuid()::text"))
```

### Timestamps
```python
created_at = sa.Column(sa.TIMESTAMP(timezone=True), server_default=sa.func.now(), nullable=False)
updated_at = sa.Column(sa.TIMESTAMP(timezone=True), server_default=sa.func.now(), onupdate=sa.func.now(), nullable=False)
```

### Indexes
- Always create indexes for foreign key columns
- Use `gin_trgm_ops` for text search columns (requires `pg_trgm` extension)
- Use GIN indexes for JSONB and array columns

### ENUMs
```python
status_enum = sa.Enum("pending", "processing", "done", "failed", name="capture_status")
mission_status_enum = sa.Enum("active", "ended", name="mission_status")
mood_enum = sa.Enum("great", "good", "meh", "bad", "awful", name="mood")
energy_enum = sa.Enum("high", "medium", "low", name="mental_energy")
```

## Core Tables Reference

### users
Managed by `fastapi-users`. Contains email, hashed_password, is_active, etc.

### library_entries
```sql
CREATE TABLE library_entries (
    id SERIAL PRIMARY KEY,
    public_id VARCHAR UNIQUE NOT NULL,
    user_id INTEGER REFERENCES users(id) NOT NULL,
    game_title VARCHAR NOT NULL,
    igdb_id INTEGER,
    cover_url VARCHAR,
    genres TEXT[],
    status VARCHAR DEFAULT 'backlog',
    notes TEXT,
    next_action VARCHAR,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

### missions
```sql
CREATE TABLE missions (
    id SERIAL PRIMARY KEY,
    public_id VARCHAR UNIQUE NOT NULL,
    user_id INTEGER REFERENCES users(id) NOT NULL,
    library_entry_id INTEGER REFERENCES library_entries(id) NOT NULL,
    status VARCHAR DEFAULT 'active',
    briefing_text TEXT,
    debrief_text TEXT,
    debrief_mood VARCHAR,
    debrief_energy VARCHAR,
    started_at TIMESTAMPTZ DEFAULT NOW(),
    ended_at TIMESTAMPTZ,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

### captures
```sql
CREATE TABLE captures (
    id SERIAL PRIMARY KEY,
    public_id VARCHAR UNIQUE NOT NULL,
    user_id INTEGER REFERENCES users(id) NOT NULL,
    capture_type VARCHAR NOT NULL,  -- 'voice', 'photo', 'text'
    raw_input TEXT,
    transcript TEXT,
    extracted_data JSONB,
    status VARCHAR DEFAULT 'pending',
    file_path VARCHAR,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

## When to use manual migration (no autogenerate)
- Seed data
- Enum value changes
- Custom indexes (partial, expression, GIN trigram)
- Rename columns/tables (autogenerate creates drop+create)
- PostgreSQL extensions
